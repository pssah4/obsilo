Memory & Chat History

Obsilo remembers your conversations and learns your preferences over time. Chat history stores every conversation for browsing and restore. The memory system automatically extracts knowledge from your conversations and personalizes the agent across sessions.

Privacy first All memory data is stored locally in your plugin directory. Memory extraction uses a dedicated model you choose (ideally a small, cheap model like Haiku). No data leaves your machine except for the LLM API calls you configure.

Chat History

Chat history is enabled by default. Every conversation is saved automatically as a JSON file inside the plugin directory (.obsidian/plugins/obsilo-agent/history/).

History Panel

Click the history icon in the sidebar header (left of the New Chat button) to open the history panel. It slides in from the left and shows your conversations grouped by date:

  • Today / Yesterday / This Week / Older
  • Each entry shows the conversation title, date, and message count
  • Use the text filter at the top to search through titles
  • Hover over an entry to reveal the delete button

Restoring a Conversation

Click any conversation in the history panel to restore it. The full message history is loaded back into the sidebar — including assistant messages, tool calls, and the internal conversation state. You can continue the conversation exactly where you left off.

Auto-Titling

After the first assistant response, Obsilo generates a short title (3-8 words) for the conversation using the active model. This title appears in the history panel and helps you find conversations later.

Memory System

The memory system has three tiers:

  • Chat history — full conversation logs stored as JSON files for browsing and restore
  • Session memory — a summary of each conversation, stored as a Markdown file
  • Long-term memory — your profile, agent identity, active projects, behavioral patterns, and task learnings, distilled from session summaries

How It Works

  1. You have a conversation with the agent as usual
  2. When the conversation ends (new chat or close), if it had enough messages (configurable, default: 6), a background extraction job is queued
  3. The extraction model (a separate, cheap model you configure) processes the queue:
    • First, it creates a session summary (key topics, decisions, preferences observed)
    • Then, it checks if any new facts should be promoted to long-term memory (your profile, projects, or patterns)
  4. Next time you start a conversation, the agent has your profile, projects, and patterns in its system prompt — plus relevant session summaries retrieved via semantic search
Zero impact on chat speed Memory extraction runs entirely in the background after the conversation ends. It never blocks or slows down your active conversation. If Obsidian closes before extraction finishes, the queue is processed on next startup.

Memory Files

Memory is stored as editable Markdown files in the global directory ~/.obsidian-agent/memory/ (shared across all vaults):

FileContainsLoaded into context
soul.mdAgent identity, personality traits, communication valuesAlways (system prompt)
user-profile.mdYour name, language, communication style, agent preferencesAlways (system prompt)
projects.mdActive projects, goals, tech stackAlways (system prompt)
patterns.mdBehavioral patterns, common refinements, workflow habitsAlways (system prompt)
learnings.mdTask learnings and insights from past interactionsAlways (system prompt)
knowledge.mdDomain expertise, specialized knowledgeOn demand (semantic search)
sessions/*.mdPer-conversation summaries with YAML frontmatterOn demand (semantic search)

You can edit these files directly — they are plain Markdown. Changes take effect on the next conversation.

Session Context Retrieval

When you start a new conversation, Obsilo uses your first message to semantically search past session summaries. The top 3 most relevant summaries are injected into the system prompt as context. This means the agent can reference previous conversations when relevant — without you having to remind it.

Onboarding

The first time you use Obsilo (when no user profile exists), the agent starts an onboarding conversation. It naturally asks about your name, language preference, vault usage, and communication style. The answers are extracted into your user profile after the conversation ends.

You can re-run onboarding at any time from Settings → Memory → Re-run Onboarding.

Settings

All memory settings are under Settings → Agent Behaviour → Memory:

SettingDefaultDescription
Enable memoryOnMaster toggle for the memory system
Auto-extract sessionsOnAutomatically create session summaries when conversations end
Auto-update long-termOnPromote new facts from sessions to long-term memory files
Memory model(none)Dropdown to pick the model for extraction. Choose a small, fast model (e.g., Haiku) for cost efficiency
Extraction threshold6Minimum message count before a conversation triggers extraction (slider: 2-20)
View memory files-Opens the memory directory in Obsidian
Reset all memory-Deletes all memory files (with confirmation)
Tip You must select a memory model before extraction works. If no model is selected, conversations are saved to history but no memory extraction runs.
