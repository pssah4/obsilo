MCP Servers

The Model Context Protocol (MCP) lets you connect external tools and data sources to Obsilo Agent. MCP servers expose tools that the agent can call just like built-in tools.

What is MCP?

MCP is an open protocol for connecting AI models to external tools and resources. An MCP server is a small program that exposes a set of tools (with JSON schemas) and executes them when called. Obsilo connects to MCP servers and makes their tools available to the agent via the use_mcp_tool tool.

Examples of what MCP servers can provide:

  • Calendar and task management (Google Calendar, Todoist)
  • Code execution and analysis
  • Database access
  • File system operations outside the vault
  • External API integrations (GitHub, Slack, Notion)
  • Custom automation scripts

Supported Transports

TransportHow it worksUse case
stdioObsilo launches a local process and communicates via stdin/stdoutLocal tools, scripts, CLI programs
SSEConnects to a remote server via Server-Sent Events (HTTP GET)Remote services, hosted tools
streamable-httpBidirectional HTTP streamingModern remote MCP servers

Adding an MCP Server

Go to Settings → Agent Behaviour → MCP and click Add Server. Fill in the server configuration:

stdio Server (Local Process)

{
  "type": "stdio",
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/me/Documents"],
  "env": {
    "MY_SECRET": "value"
  },
  "timeout": 30000,
  "alwaysAllow": ["read_file", "list_directory"]
}

SSE Server (Remote)

{
  "type": "sse",
  "url": "https://my-mcp-server.example.com/sse",
  "headers": {
    "Authorization": "Bearer my-token"
  },
  "timeout": 30000
}

Streamable HTTP Server

{
  "type": "streamable-http",
  "url": "https://my-mcp-server.example.com/mcp",
  "headers": {
    "Authorization": "Bearer my-token"
  }
}

Configuration Fields

FieldTypeDescription
typestringTransport: "stdio", "sse", or "streamable-http"
commandstring(stdio only) The command to run (e.g., "node", "npx", "python")
argsstring[](stdio only) Command arguments
envobject(stdio only) Environment variables injected into the process
urlstring(sse/http) Server URL
headersobject(sse/http) HTTP headers (e.g., Authorization)
timeoutnumberConnection timeout in ms (default: 30000)
alwaysAllowstring[]Tool names on this server that never require user approval
disabledbooleanTemporarily disable this server without removing it

Popular MCP Servers

Filesystem

Access files outside the vault:

name: "filesystem"
type: stdio
command: npx
args: ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allow"]

Fetch (Web)

Enhanced web fetching with cookie support:

name: "fetch"
type: stdio
command: npx
args: ["-y", "@modelcontextprotocol/server-fetch"]

GitHub

Access GitHub repositories, issues, and PRs:

name: "github"
type: stdio
command: npx
args: ["-y", "@modelcontextprotocol/server-github"]
env:
  GITHUB_PERSONAL_ACCESS_TOKEN: "ghp_..."

Sqlite

Query and update SQLite databases:

name: "sqlite"
type: stdio
command: npx
args: ["-y", "@modelcontextprotocol/server-sqlite", "--db-path", "/path/to/db.sqlite"]

Tool Discovery

When Obsilo connects to an MCP server, it automatically queries the server's tool list. All discovered tools are then listed in the agent's system prompt so the agent knows they exist and how to use them. Tools appear as:

Connected servers and their tools:
  - filesystem: read_file — Read complete contents of a file
  - filesystem: list_directory — List files in a directory
  - github: create_issue — Create a GitHub issue

Per-Mode Server Whitelisting

You can restrict which MCP servers are available in specific modes:

  1. Go to Settings → Modes → (edit a mode) → Allowed MCP Servers
  2. Select which servers this mode can use
  3. If none are selected, all configured servers are allowed

This lets you give a research mode access to a web-search MCP server while restricting other modes from it.

Always-Allow Lists

The alwaysAllow field on a server config lists tool names that never require approval. For example, if you always trust reads from a filesystem server:

"alwaysAllow": ["read_file", "list_directory"]

All other tools from that server still follow normal approval rules.

Security Considerations

MCP servers run with system permissions stdio MCP servers run as local processes with the same permissions as Obsidian. Only connect servers you trust. The MCP tool requires approval by default — enable auto-approval only for trusted servers.
  • Review what each server's tools can do before enabling auto-approval
  • Use env to pass API keys rather than embedding them in command args
  • Use per-mode whitelisting to limit which modes can access powerful tools
