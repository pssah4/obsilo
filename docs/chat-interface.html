Chat Interface

Everything you do in Obsilo starts in the sidebar. Here's how to get the most out of it — from sending your first message to attaching files, using autocomplete, and reading the activity log.

Overview

The sidebar has three main zones:

  • Header — mode selector, model selector, new chat button
  • Messages area — conversation history with real-time streaming
  • Input area — text input with toolbar (attachments, tools, send)

Sending Messages

Type your message in the input area at the bottom of the sidebar.

  • Enter — send the message (default)
  • Shift+Enter — insert a newline
  • Ctrl/Cmd+Enter — alternative send key (configurable in Settings → Interface)

You can change the send key behavior in Settings → Interface → "Press Enter to send".

Context Injection

Active File Context

Enable "Auto-add active file context" in Settings → Interface to automatically include the note you're currently viewing in every message. A badge in the input area shows the filename. This lets you ask questions like "Summarize this note" or "Find related notes" without having to attach the file manually.

Attaching Files

Type @ in the input to open the Vault File Picker — a live-search list of all your vault files. Select one (or several) to include their content as context. Attached files appear as chips in the input area and can be removed individually. This is useful when you want the agent to reference a specific note without switching to it.

Autocomplete

Two shortcut triggers are available in the input:

  • / — opens a list of your workflows and custom prompts. Select one to expand it as a message template. For example, type /daily-review to run your review workflow.
  • @ — opens the vault file picker to attach notes as context.

Activity Block

When the agent is working, an expandable activity block appears below the spinner. It shows every tool call in real time — the tool name, its inputs, and the result (success or error). You can expand it at any time while the agent is still running.

Each tool call shows:

  • Tool name in a monospace badge
  • Key parameters (e.g., the file path for read_file)
  • Diff badge for write operations: +N / -N lines changed
  • Result shown on expansion

Task Checklist

When the agent calls update_todo_list to plan a complex task, a checklist card appears in the conversation. It updates live as the agent completes steps:

  • [ ] — pending
  • [~] — in progress (highlighted)
  • [x] — complete

Approval Flow

When the agent wants to create, edit, or delete a file, an approval card appears:

  • Tool name and file path — what will change
  • Diff preview — added (green) and removed (red) lines for edits
  • Approve — execute the operation and continue
  • Reject — block this operation (the agent will be told it was rejected)
  • Auto-approve — approve this operation type for the rest of the session
Tip Configure permanent auto-approval in Settings → Permissions to skip the approval card for operations you always trust (e.g., read operations are auto-approved by default).

Token Usage

After each response, a token badge shows the cumulative input and output tokens used in the conversation. This helps you track costs and understand when context condensing might be useful.

Mode and Model Selector

The chat header has two selectors:

  • Mode selector (left) — switch between Ask, Agent, and custom modes. The switch takes effect for the next message.
  • Model selector (right) — switch the AI model. Shows all enabled models.

The agent can also switch modes mid-conversation with the switch_mode tool if the task calls for it (requires approval).

Chat Controls

  • New Chat (header button) — clears the conversation and starts fresh
  • Stop (appears while agent is running) — aborts the current request
  • Regenerate — re-sends the last user message, discarding the previous response
  • Save Chat — saves the current conversation to a JSON file in your vault (if chat history folder is configured)

Chat History

Chat history is enabled by default. Every conversation is automatically saved and can be restored from the history panel. Click the history icon in the sidebar header to browse past conversations grouped by date.

For full details on chat history, memory, and personalization, see Memory & Chat History.

Tool Picker (Session Overrides)

Click the wrench icon in the input toolbar to open the Tool Picker popover. This lets you temporarily override which tools, skills, or workflows are active for the current session — without changing your permanent settings.
