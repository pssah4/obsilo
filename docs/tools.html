Tools Reference

Complete reference for all 30+ tools available to Obsilo Agent. Tools are grouped by category, which maps to the tool group permission system.

Read Tools

These tools are read-only and are in the read tool group. Auto-approved by default.

read
read_file
Read the complete content of a file. Always call this before editing an existing file.
read
list_files
List files and folders in a directory. Use "/" for the vault root. Optional recursive parameter.
read
search_files
Full-text and regex search across files. Returns matching lines with file path and line number. Supports file_pattern glob filter.

Parameters: read_file

ParameterTypeRequiredDescription
pathstringyesVault-relative path to the file (e.g., "Projects/plan.md")

Parameters: list_files

ParameterTypeRequiredDescription
pathstringyesDirectory path. Use "/" for vault root
recursivebooleannoInclude subdirectory contents (default: false)

Parameters: search_files

ParameterTypeRequiredDescription
pathstringyesDirectory to search in
patternstringyesText or regex pattern to search for
file_patternstringnoGlob pattern to filter files (e.g., "*.md")

Vault Intelligence Tools

These tools are read-only metadata and search operations. In the vault tool group. Auto-approved by default.

vault
get_vault_stats
Returns a vault overview: total notes, folder tree, top tags, recently modified files. No parameters needed.
vault
get_frontmatter
Read all YAML frontmatter fields of a note (tags, aliases, dates, custom properties).
vault
search_by_tag
Find all notes with given tags. Supports AND or OR matching. Tags with or without # both work.
vault
get_linked_notes
Get forward links and backlinks for a note. Direction: "both" (default), "forward", or "backlinks".
vault
get_daily_note
Read a daily note. offset=0 (today), -1 (yesterday), 1 (tomorrow). create=true creates it if missing.
vault
open_note
Open a note in the Obsidian editor view. Use after creating or editing a note to bring it into focus.
vault
semantic_search
Hybrid semantic + keyword search. Finds notes by meaning. Requires Semantic Index to be built. Supports folder, tag, and date filters.
vault
query_base
Query an Obsidian Bases (.base) file. Returns the notes matching the view's filter conditions.

Parameters: semantic_search

ParameterTypeRequiredDescription
querystringyesNatural language search query
top_knumbernoMax results to return (default: 8, max: 20)
folderstringnoRestrict results to notes in this folder (prefix match)
tagsstring[]noOnly return notes that have ANY of these tags
sincestringnoOnly return notes modified on or after this date (ISO format: "2025-01-01")

Writing & Editing Tools

These tools modify vault files and require approval (or auto-approval). In the edit tool group.

edit
write_file
Create a new file or completely replace an existing file's content. Preferred for new notes or full rewrites.
edit
edit_file
Replace a specific string in an existing file. Safer than write_file for targeted changes — preserves surrounding content.
edit
append_to_file
Append content to the end of a file. Ideal for daily notes, logs, and additive entries. Optional separator before appended content.
edit
update_frontmatter
Set or update YAML frontmatter fields. Preserves all existing fields. Creates frontmatter if none exists.
edit
create_folder
Create a folder, including all parent folders in the path.
edit
delete_file
Move a file or empty folder to the Obsidian trash (recoverable). Does not permanently delete.
edit
move_file
Move or rename a file or folder. Updates wikilinks automatically via Obsidian's link resolver.

Parameters: edit_file

ParameterTypeRequiredDescription
pathstringyesVault-relative file path
old_strstringyesExact string to replace (including whitespace). Must match exactly.
new_strstringyesReplacement string
expected_replacementsnumbernoIf set, fails if the count of replacements doesn't match (safety check)

Advanced Vault Tools

edit
generate_canvas
Create an Obsidian Canvas (.canvas) file visualizing notes and their connections. Supports folder, tag, backlinks, and manual file list modes.
edit
create_base
Create an Obsidian Bases (.base) database view file with configurable filters, columns, and sort order.
edit
update_base
Add or replace a named view in an existing Bases file.

Parameters: generate_canvas

ParameterTypeRequiredDescription
output_pathstringyesPath for the output .canvas file
modestringyes"folder" | "tag" | "backlinks" | "files"
sourcestringnoFor folder/tag mode: folder path or tag name
filesstring[]noFor files mode: explicit list of file paths
max_notesnumbernoMax notes to include (default: 20)
draw_edgesbooleannoDraw edges for wikilink connections (default: true)

Web Tools

Require Web Tools to be enabled in Settings → Web Search. In the web tool group.

web
web_fetch
Fetch a URL and return its content as Markdown. Handles pagination via startIndex. Max 20,000 chars by default.
web
web_search
Search the web and return titles, URLs, and snippets. Requires Brave or Tavily API key. Follow up with web_fetch to read full pages.

Agent Control Tools

These tools control the agent loop. In the agent tool group.

agent
ask_followup_question
Ask the user a clarifying question with optional answer choices. Pauses the loop until the user responds.
agent
attempt_completion
Signal that the task is complete. Call this AFTER streaming your full response as text. The result field is an internal log entry, not shown to the user.
agent
update_todo_list
Publish a task plan as a visible checklist. Use only for tasks with 3+ distinct steps. Statuses: pending ([ ]), in-progress ([~]), done ([x]).
agent
switch_mode
Switch the active mode mid-conversation. Requires user approval by default.
agent
new_task
Spawn a sub-agent in a fresh context. Available in Agent mode only. Pass all necessary context in the message — the sub-agent cannot see this conversation.

Parameters: new_task

ParameterTypeRequiredDescription
modestringyes"agent" or "ask" (or any custom mode slug)
messagestringyesFull task description with all necessary context

MCP Tool

mcp
use_mcp_tool
Call a tool on a configured MCP server. Tool names and descriptions are dynamically listed in the system prompt from connected servers.

Parameters: use_mcp_tool

ParameterTypeRequiredDescription
server_namestringyesName of the configured MCP server
tool_namestringyesName of the tool to call on that server
argumentsobjectyesTool arguments as key-value pairs (schema from server)
