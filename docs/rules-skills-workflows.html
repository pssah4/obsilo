Rules, Skills & Workflows

Tailor Obsilo to your vault, writing style, and daily routines. Set permanent conventions the agent always follows, activate context-specific instructions for particular task types, and invoke step-by-step procedures on demand — all without touching the agent's core settings.

MechanismWhen it appliesBest for
RulesAlways (every message)Permanent conventions the agent must always follow
SkillsWhen keywords match user messageContext-specific instructions for particular task types
WorkflowsWhen user types /workflow-nameStep-by-step procedures invoked on demand
Custom PromptsWhen user types /prompt-slugMessage templates with dynamic variables

Rules

Rules are Markdown files that are always injected into the agent's system prompt as a RULES section. The agent must follow them in every response.

Location

<vault>/.obsidian-agent/rules/

Any .md or .txt file in this folder is treated as a rule file.

Example Rules

Create .obsidian-agent/rules/formatting.md:

## Formatting Conventions

- Always use ISO 8601 dates (YYYY-MM-DD) in frontmatter, not natural language dates
- Use [[wikilinks]] for internal note references, not markdown links
- Every new note must have at least: title, tags, and created date in frontmatter
- Headers: ## for main sections, ### for subsections — never use # (reserved for note title)
- Tag style: lowercase, hyphenated — use #machine-learning not #MachineLearning

Create .obsidian-agent/rules/language.md:

Always respond in German. When creating note content, use German unless the
existing note content is in another language.

Managing Rules

Go to Settings → Agent Behaviour → Rules:

  • Create new rule files with the inline editor
  • Toggle individual rules on/off (disabled rules are not injected)
  • Edit existing rule files
  • Delete rule files

New rules are enabled by default. Disable them without deleting by toggling them off.

Size limit Each rule file is capped at 50,000 characters to prevent unintentionally large system prompts.

Skills

Skills are modular instruction sets that the agent can apply for specific task types. Unlike rules (which are always active), skills are automatically matched to each message by keyword overlap.

Location

<vault>/.obsidian-agent/skills/
  meeting-notes/
    SKILL.md
  literature-review/
    SKILL.md
  project-planning/
    SKILL.md

Each skill is a folder containing a SKILL.md file.

SKILL.md Format

---
name: meeting-notes
description: taking meeting notes, capturing action items, agenda, attendees, follow-up
---

When writing meeting notes, follow this structure:

## Standard Meeting Note Template

Use these sections:
- **## Attendees** — list all participants
- **## Agenda** — planned topics
- **## Notes** — what was discussed
- **## Decisions** — explicit decisions made
- **## Action Items** — format: `- [ ] @person: task (due: YYYY-MM-DD)`

Always link to related projects with [[wikilinks]].
Create a separate action item note if there are 5+ action items.

The frontmatter is required:

  • name — short identifier (lowercase, hyphens)
  • description — words here are used for keyword matching against the user's message

Keyword Matching

Before each message, Obsilo checks if any skill's description words overlap with words in the user message (3+ character words, case-insensitive). Matching skills are inlined into the system prompt as an <available_skills> block.

For example, if the user writes "Write meeting notes for today's planning call", the meeting-notes skill (description: "taking meeting notes, capturing action items, agenda...") matches on "meeting", "notes", "planning".

Forced Skills

In Settings → Modes → (edit mode) → Forced Skills, you can specify skills that are always injected for a specific mode — regardless of keyword matching. Useful when a mode should always apply a particular style or set of conventions.

Managing Skills

Go to Settings → Agent Behaviour → Skills to view, edit, and manage your skills. You can also create skill files directly in your vault.

Workflows

Workflows are step-by-step instruction templates stored as Markdown files. They are invoked with a slash command typed at the start of a message.

Location

<vault>/.obsidian-agent/workflows/

Workflow Format

A workflow is just a plain Markdown file. The filename (without extension) becomes the slash command slug.

File: .obsidian-agent/workflows/daily-review.md

Review today's daily note and provide a structured end-of-day summary:

1. Read today's daily note using get_daily_note
2. List all open tasks (unchecked checkboxes)
3. List all completed tasks
4. Identify any notes that were mentioned or created today
5. Summarize the key themes or topics from today
6. Suggest 3 priorities for tomorrow based on open items

Format the output as a structured Markdown report.

Using Workflows

Type the slash command at the start of a message:

/daily-review
/daily-review Focus especially on the project meeting notes
/literature-review Find connections to my existing notes on the topic

The workflow content is prepended to the message as explicit instructions. Any text after the slash command is appended after the workflow instructions.

Forced Workflow (Per Mode)

In Settings → Modes → (edit mode) → Forced Workflow, you can attach a workflow that automatically applies to every message in that mode (unless the message itself starts with /). This is useful for modes with a very specific operating procedure.

Managing Workflows

Go to Settings → Agent Behaviour → Workflows to create, edit, and toggle workflows. Disabled workflows don't appear in autocomplete and aren't applied even if typed.

Custom Prompts

Custom prompts are slash-command templates with variable substitution. Unlike workflows (which are fixed instruction sets), custom prompts can include dynamic content from the current context.

Variables

VariableValue
{{userInput}}The text the user typed after the slash command
{{activeFile}}The path of the currently open file in Obsidian

Example

Prompt: "Summarize Note", slug: summarize

Please create a concise summary of {{activeFile}}.

{{userInput}}

Format as:
- 3-5 bullet points of key insights
- 1 sentence "bottom line"
- Suggested tags based on the content

Usage: /summarize Focus on the methodology section

Managing Prompts

Go to Settings → Agent Behaviour → Prompts. Create, edit, enable/disable, and delete prompts. You can optionally restrict a prompt to a specific mode (it only shows in autocomplete and works in that mode).
