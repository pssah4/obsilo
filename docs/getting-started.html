Getting Started

Obsilo is in early beta and open for testing. It has not been published as Community Plugin yet, hence you'll need to use one of the two other options described below.
Install Obsilo Agent, connect your first AI model, and run your first task in under 5 minutes.

What you'll be able to do Ask questions about your vault, create and edit notes with AI assistance, research topics on the web and save the results, organize notes in bulk — all from a sidebar inside Obsidian, with a diff preview and approval step before anything changes.

Requirements

ObsidianVersion 1.4.0 or later
PlatformDesktop only (macOS, Windows, Linux). Not available on mobile.
Node.js18 or later (for building from source)
API KeyAnthropic, OpenAI, OpenRouter, or a running Ollama instance

Installation

Install via BRAT (Recommended)

BRAT (Beta Reviewers Auto-update Tool) lets you install Obsilo Agent directly from GitHub without building from source, and keeps it up to date automatically.

  1. Install BRAT

    In Obsidian: Settings → Community Plugins → Browse → search for "BRAT" → Install and Enable.

  2. Add Obsilo Agent

    Open BRAT settings → Add Beta Plugin → paste:

    https://github.com/pssah4/obsilo-public

    Click Add Plugin. BRAT downloads and installs the latest release.

  3. Enable the plugin

    Go to Settings → Community Plugins → scroll to "Obsilo Agent" → toggle on.

Manual Installation (from source)

  1. Clone the repository 
    git clone https://github.com/pssah4/obsilo-public.git
cd obsidian-agent
  2. Install dependencies and build 
    npm install
npm run build
    This produces main.js in the project root.
  3. Copy to your vault

    Create the plugin directory and copy the three required files:

    mkdir -p "/path/to/your-vault/.obsidian/plugins/obsidian-agent"
cp main.js styles.css manifest.json \
   "/path/to/your-vault/.obsidian/plugins/obsidian-agent/"
  4. Enable in Obsidian

    Open Obsidian → Settings → Community Plugins → scroll to "Obsilo Agent" → toggle it on.

    Note If you don't see the plugin, make sure "Safe Mode" is turned off in Community Plugins settings.
Development mode Run npm run dev to watch for changes and hot-deploy on every save. Set the target vault path in esbuild.config.mjs (VAULT_PLUGIN_DIR) to auto-deploy on each build.

First Setup

1. Add an AI Model

Open Obsidian Settings → Obsilo Agent → Providers → Models.

You'll see pre-configured built-in models. Click Edit on any model to add your API key and enable it:

ProviderWhere to get keyRecommended model
Anthropicconsole.anthropic.comClaude Sonnet 4.5
OpenAIplatform.openai.comGPT-4o
OpenRouteropenrouter.aiLlama 3.2 3B (free tier)
OllamaRun locally: ollama servellama3.2 or qwen2.5:7b

After entering the key, click Test Connection to verify it works. Then enable the model to make it available in the chat.

2. Open the Sidebar

Click the Obsilo Agent icon in the Obsidian ribbon (left or right edge of the window). If you don't see it, use the Command Palette (Ctrl/Cmd+P) and search for "Open Agent Sidebar".

The sidebar defaults to the right panel. You can move it to the left in Settings → Interface → Sidebar Position.

3. Your First Task

The sidebar opens in Ask mode by default — a read-only assistant that answers questions about your vault without touching any files. Try:

  • "What are my most recently modified notes?"
  • "Find all notes tagged with #project"
  • "What do I know about [any topic in your vault]?"

Switch to Agent mode using the mode selector in the chat header when you want to create, edit, or reorganize. The agent will show you a diff and ask for your approval before changing any file. Try:

  • "Create a weekly review template with sections for goals, accomplishments, and next steps"
  • "Add a 'status' frontmatter field to all notes in my Projects folder"
  • "Research [topic] on the web and save a summary note to my Research folder"

4. Set Up Semantic Search (Optional but Recommended)

Semantic search lets the agent find notes by meaning rather than just keywords. It runs entirely locally — no cloud service needed.

  1. Go to Settings → Obsilo Agent → Semantic Index
  2. Toggle "Enable Semantic Index" on
  3. Choose an embedding model. The default Xenova/all-MiniLM-L6-v2 runs locally and requires no API key
  4. Click Build Index and wait for indexing to complete (the first build may take a few minutes)

Once built, the semantic_search tool is available to the agent in all modes.
