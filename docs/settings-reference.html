Settings Reference

Complete reference for every setting in Obsilo Agent. Open via Obsidian Settings → Obsilo Agent.

Providers

Models

SettingDescription
Active Models listList of all configured models. Each entry shows provider, name, and enabled status.
Add Custom ModelAdd a new model with any provider type. Set name, provider, API key, base URL, temperature, max tokens.
Global Custom InstructionsText appended to the system prompt for ALL modes. Use for universal formatting rules, language preferences, etc.
Default ModelThe model selected by default in new chats. Set by enabling and selecting a model.

Embeddings

SettingDescription
Embedding ModelModel used to generate vectors for semantic search. Changing this requires rebuilding the index.
Add Custom Embedding ModelAdd a Xenova/transformers model by name or an API-based embedding model.

Agent Behaviour

Modes

SettingDescription
Built-in modesAsk and Agent modes are always present. Their role definitions can be viewed but not edited.
Custom Modes listUser-created modes (vault or global scope).
New ModeCreate a custom mode with slug, name, icon, description, role definition, custom instructions, tool groups, and scope.
Per-mode ModelOverride the default model for a specific mode.
Per-mode ToolsRestrict available tools to a subset of the mode's tool groups.
Per-mode MCP ServersWhitelist specific MCP servers for this mode.
Forced SkillsSkills always injected for this mode regardless of keyword matching.
Forced WorkflowWorkflow applied to every message in this mode (unless message starts with /).

Permissions

SettingDefaultDescription
Enable Auto-ApprovalOffMaster toggle. When off, every operation requires manual approval.
Show quick-toggle bar in chatOnDisplay permission toggles at the top of the chat view.
Read operationsAuto-approvedread_file, list_files, search_files, and vault intelligence tools.
Note editsManualwrite_file, edit_file, append_to_file, update_frontmatter.
Vault changesManualcreate_folder, delete_file, move_file.
Web operationsManualweb_fetch, web_search.
MCP toolsManualuse_mcp_tool.
Mode switchingManualswitch_mode.
SubtasksManualnew_task (spawn sub-agents).
QuestionsAuto-approvedask_followup_question (shows question card directly without approval step).
Todo updatesAuto-approvedupdate_todo_list.

Loop

SettingDefaultDescription
Consecutive Mistake Limit3Stop the agent after N consecutive tool errors. 0 = disabled.
Rate Limit (ms)0Minimum milliseconds between API iterations. 0 = no limit. Use to avoid rate limit errors.
Context CondensingOffAutomatically summarize conversation when estimated tokens exceed the threshold.
Condensing Threshold80%% of the model's context window at which to trigger condensing (50–95).
Power Steering Frequency0Inject a mode-role reminder every N iterations to keep the agent on task. 0 = disabled.

Rules

SettingDescription
Rules listAll .md and .txt files in .obsidian-agent/rules/. Toggle each on/off.
New RuleCreate a new rule file with an inline editor.
Edit / DeleteManage existing rule files.

Workflows

SettingDescription
Workflows listAll .md and .txt files in .obsidian-agent/workflows/. Toggle each on/off. Shows the slash command slug.
New WorkflowCreate a new workflow file with an inline editor.
Edit / DeleteManage existing workflow files.

Skills

SettingDescription
Skills listAll discovered SKILL.md files. Shows name, description (used for keyword matching), and match count.
New SkillCreate a new skill folder and SKILL.md file.
Edit DescriptionUpdate the keyword-matching description without editing the skill file directly.

Prompts

SettingDescription
Custom Prompts listAll user-defined slash-command prompt templates.
NameDisplay name for the prompt.
SlugThe /slug used to invoke the prompt.
ContentPrompt template text. Supports {{userInput}} and {{activeFile}} variables.
EnabledToggle: shown in autocomplete and functional when enabled.
Mode RestrictionOptional: only show this prompt in a specific mode.

MCP

SettingDescription
MCP Servers listAll configured servers with name, type, and connection status.
Add ServerAdd a new MCP server configuration.
Active Servers whitelistRestrict which configured servers are globally active. Empty = all allowed.
Connect / DisconnectManually connect or disconnect a server.

Vault

Semantic Index

SettingDefaultDescription
Enable Semantic IndexOffMaster toggle for semantic search. Must be on for semantic_search tool to work.
Auto-IndexneverWhen to automatically rebuild: "startup", "mode-switch", or "never".
Auto-Index on ChangeOffRe-index files as they change. Can cause UI lag with local models on large vaults.
Build Index—Button: start a full (incremental) index build now.
Chunk Size2000Target characters per chunk. Smaller = more precise. Changing requires full rebuild.
Batch Size20Files per disk commit during indexing.
Excluded Folders—Comma-separated folders to skip (e.g., "Templates, Archive").
Index PDFsOffExtract and index text from PDF files.
HyDEOffGenerate hypothetical document before query. Costs 1 extra LLM call per search.
Storage Locationobsidian-sync"obsidian-sync" = syncs with Obsidian Sync; "local" = device-only.

Checkpoints

SettingDefaultDescription
Enable CheckpointsOnTake automatic snapshots before task write operations.
Checkpoint Timeout30sMax time to wait for a snapshot. If exceeded, task continues without checkpoint.
Auto-cleanupOnRemove old checkpoints automatically.

Chat History

SettingDefaultDescription
Chat History Folder(empty)Vault folder to save conversations to. Empty = disabled. Creates JSON files per conversation.

Web Search

SettingDefaultDescription
Enable Web ToolsOffEnable web_fetch and web_search tools.
Search Providernone"brave", "tavily", or "none" (web_search disabled if none).
Brave API Key—API key for Brave Search.
Tavily API Key—API key for Tavily Search.

Advanced

Interface

SettingDefaultDescription
Sidebar Positionright"left" or "right" — which side the Obsilo panel appears on.
Press Enter to sendOnWhen on, Enter sends the message and Shift+Enter inserts a newline. When off, Ctrl/Cmd+Enter sends.
Auto-add active file contextOnAutomatically include the currently open note as context in every message.
Show welcome messageOnShow a welcome message when the sidebar first opens.
Include current time in contextOnInject today's date and time into the system prompt.

Logs

View the operation audit trail. Logs are JSONL files at .obsidian/plugins/obsidian-agent/logs/YYYY-MM-DD.jsonl. Select a date to view that day's log. Retained for 30 days.

Backup

Browse and restore from task checkpoints. Each entry shows the task, timestamp, and files changed. Use Preview to see a diff and Restore to revert those files.

Debug

SettingDefaultDescription
Debug ModeOffEnable verbose console logging for troubleshooting. Output visible in Obsidian developer tools (Ctrl+Shift+I).
