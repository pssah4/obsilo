Providers & Models

Configure the AI models that power Obsilo. You can add multiple models from different providers and switch between them per-chat or per-mode.

Adding a Model

  1. Go to Settings → Providers → Models
  2. Find the model in the list and click Edit, or click Add Custom Model for a new one
  3. Enter your API Key
  4. Optionally adjust Temperature (0.0–1.0) and Max Tokens
  5. Click Test Connection to verify it works
  6. Toggle the model Enabled to make it available in the model selector

Provider-Specific Setup

Anthropic

  • Get your API key at console.anthropic.com
  • Key format: sk-ant-api03-...
  • No base URL needed (uses default Anthropic endpoint)

OpenAI

  • Get your API key at platform.openai.com/api-keys
  • Key format: sk-proj-... or sk-...
  • No base URL needed (uses default OpenAI endpoint)

OpenRouter

  • Get your API key at openrouter.ai/keys
  • Base URL is pre-configured: https://openrouter.ai/api/v1
  • Model IDs follow the format provider/model-name
  • Free models are available — check the OpenRouter catalog

Ollama (Local)

  1. Install Ollama from ollama.ai
  2. Pull a model: ollama pull llama3.2
  3. Start the server: ollama serve (usually starts automatically)
  4. In Obsilo: Base URL = http://localhost:11434, no API key needed
Model ID format for Ollama Use the exact model name as shown in ollama list — e.g., llama3.2, qwen2.5:7b, codellama.

LM Studio

  • Download LM Studio from lmstudio.ai
  • Start the local server in LM Studio (usually port 1234)
  • Base URL: http://localhost:1234/v1
  • Model ID: use the model name shown in LM Studio (e.g., Meta-Llama-3.1-8B-Instruct-GGUF)

Azure OpenAI

  • Base URL format: https://YOUR-RESOURCE.openai.azure.com/openai/deployments/YOUR-DEPLOYMENT
  • API Version: e.g., 2024-10-21 (required for Azure)
  • API Key: your Azure OpenAI key
  • Model ID: your deployment name in Azure

Custom (OpenAI-Compatible)

Use this for any server with an OpenAI-compatible API:

  • Set the Base URL to your server's base (e.g., http://my-server:8000/v1)
  • Set the Model ID to the model name your server expects
  • Add an API Key if your server requires one

Embedding Models

Embedding models are separate from chat models and are used for the Semantic Index. Configure them in Settings → Providers → Embeddings.

Local Embedding Models (Xenova)

These run entirely locally using ONNX via Xenova/transformers. They download once (~23-90MB) and then run offline:

ModelDimensionsLanguagesNotes

API Embedding Models

Use OpenAI's or another provider's embedding API:

ModelProviderDimensionsNotes
text-embedding-3-smallOpenAI1536High quality, requires API key
text-embedding-3-largeOpenAI3072Best quality, higher cost
Changing the embedding model requires a full index rebuild. Vectors from different models are not compatible. If you switch models, go to Settings → Semantic Index and rebuild the index.

Global Custom Instructions

In Settings → Providers → Models, you can set Global Custom Instructions. These are appended to the system prompt for every mode. Use them for universal preferences that apply regardless of what the agent is doing:

Always respond in German.
When creating frontmatter, include: title, tags, created, status.
Prefer concise responses — use bullet points over prose.
